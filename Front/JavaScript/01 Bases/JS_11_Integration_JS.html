11 Intégrer du JavaScript dans une page web

Il existe plusieurs façons d’intégrer du JavaScript dans une page HTML. Certaines sont historiques et plutôt à éviter aujourd’hui, d’autres correspondent aux bonnes pratiques modernes.

Remarque : comme pour les feuilles de style CSS, on distingue du JavaScript intégré directement dans le HTML et du JavaScript externalisé dans des fichiers .js.

1. Intégration directe dans une balise HTML (inline)

On peut écrire du JavaScript directement dans les attributs des balises HTML, via des attributs d’événements : onclick, onmouseover, onchange, etc.

Cette méthode fonctionne, mais elle est aujourd’hui déconseillée :

  • elle mélange structure (HTML) et comportement (JS) ;
  • elle rend le code difficile à maintenir et à tester ;
  • elle complique l’accessibilité (navigation clavier, lecteurs d’écran…).

Exemple (à ne pas reproduire dans du code moderne) :

<!DOCTYPE html>
<html>
<head>
  <title>Titre du fichier</title>
</head>
<body>
  <a
    href="page.html"
    title="Voir la page"
    onclick="alert('Lien cliqué !');"
  >Aller à la page</a>
</body>
<html>

Version moderne recommandée pour le même comportement (séparation HTML / JS) : 

<!DOCTYPE html>
<html>
<head>
  <title>Titre du fichier</title>
</head>
<body>
  <a
    id="lien-page"
    href="page.html"
    title="Voir la page"
  >Aller à la page</a>

  <script>
  const lien = document.getElementById("lien-page");
  lien.addEventListener("click", (event) => {
    // Exemple : on affiche un message et on laisse le lien fonctionner normalement
    console.log("Lien cliqué !");
  });
  </script>
</body>
</html>

2. Intégration via une balise <script>

On peut écrire du JavaScript à l’intérieur d’un <script> dans la page HTML. C’est ce qu’on appelle un script interne.

Exemple minimal :

<!DOCTYPE html>
<html>
<head>
  <title>Titre du fichier</title>
</head>
<body>
  <p>Bonjour ! Ouvrez la console du navigateur.</p>

  <script>
  console.log("Hello JavaScript !");
  </script>
</body>
</html>

Historiquement, on plaçait souvent les scripts dans le <head>, ce qui bloquait le chargement de la page en attendant que les scripts soient téléchargés et exécutés.

En pratique, il est préférable de :

  • soit placer les scripts à la fin de <body> ;
  • soit les charger dans le <head> avec l’attribut defer (scripts différés).

Exemple avec defer

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>Exemple avec defer</title>
  <!-- Le script est téléchargé en parallèle, puis exécuté une fois le DOM prêt -->
  <script src="app.js" defer></script>
</head>
<body>
  <h1>Page avec script différé</h1>
  <button id="btn-test">Tester</button>
</body>
</html>

Dans app.js :

const bouton = document.getElementById("btn-test");
bouton.addEventListener("click", () => {
  console.log("Bouton cliqué !");
});

Grâce à defer, on n’a pas besoin d’attendre DOMContentLoaded : le script s’exécute une fois le DOM prêt, dans l’ordre des balises <script>.

3. Intégration via un fichier JavaScript externe

C’est la méthode recommandée pour des projets réels :

  • on sépare le HTML (structure), le CSS (présentation) et le JS (comportement) ;
  • les fichiers .js sont mis en cache par le navigateur ;
  • on évite de dupliquer le même script sur plusieurs pages : un seul fichier partagé.

L’appel du fichier .js se fait via une balise <script> avec l’attribut src, de préférence avec defer : 

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>Titre du fichier</title>
  <!-- Fichier JavaScript externe -->
  <script src="assets/js/monscript.js" defer></script>
</head>
<body>
  <a
    href="page.html"
    title="Voir la page"
    id="lien-exemple"
  >Aller à la page</a>
</body>
</html>

L’attribut src accepte un chemin relatif (ex : assets/js/monscript.js) ou une URL complète (par exemple un CDN : https://cdn.example.com/lib.js).

Option moderne : scripts de type module

Pour profiter des imports/exports ES6 (modules JavaScript), on peut utiliser type="module" : 

<script type="module" src="assets/js/main.js"></script>

Dans main.js :

import { initialiserUI } from "./ui.js";

initialiserUI();

Les modules sont automatiquement exécutés en mode strict et sont différés (comportement proche de defer).

4. Chargement du DOM et événement DOMContentLoaded

Si tu ne peux pas (ou ne veux pas) utiliser defer, tu peux attendre que le DOM soit prêt avant de manipuler les éléments HTML : 

<!DOCTYPE html>
<html>
<head>
  <title>DOM prêt</title>
  <script>
  document.addEventListener("DOMContentLoaded", () => {
    const titre = document.querySelector("h1");
    titre.textContent = "DOM chargé !";
  });
  </script>
</head>
<body>
  <h1>Chargement en cours...</h1>
</body>
</html>

5. Exercices : du code intégré à l’externe

Le but est d’externaliser le code JavaScript dans un fichier .js, puis de tester l’ajout de contenu HTML via JavaScript.

Exercice 1 – Version « historique » (pour comprendre)

  1. Copie le code suivant dans ton éditeur et enregistre-le sous exercice1.html. 

    <!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>Mon 1er script</title>
  <!-- Script intégré (peu recommandé, mais utile pour comprendre) -->
  <script>
  alert("Coucou !");
  console.log("Ah que Coucou !");
  document.write("Coucou beuh !");
  </script>
</head>
<body>
  <!-- Le contenu ci-dessous sera en partie remplacé par document.write() -->
  <p>Rien à ajouter</p>
</body>
</html>

    Ouvre ce fichier dans ton navigateur. Observe : alert, console et document.write. Remarque ce qui arrive au contenu de <body>.

  2. Crée un fichier JavaScript nommé exercice1.js.
  3. Déplace le code JavaScript présent dans la balise <script> vers exercice1.js, puis modifie le HTML pour importer ce fichier en externe (avec defer de préférence).
  4. Modifie l’appel à document.write() pour y inclure du HTML, par exemple : <h1>Titre ajouté</h1>. Que constates-tu sur le rendu final de la page ?
  5. Modifie alert() pour y passer du « pseudo-HTML » (par exemple "<strong>Texte</strong>"). Est-ce que l’alerte interprète le HTML ?

Important : document.write() est aujourd’hui considéré comme une pratique obsolète. Il peut remplacer le contenu de la page ou casser le DOM s’il est utilisé après le chargement initial. Dans le code moderne, on lui préfère les manipulations du DOM (innerHTML, appendChild, etc.).

Exercice 2 – Version moderne (recommandée)

  1. À partir du même HTML, remplace l’appel à document.write() par une modification d’un élément existant du DOM :
    • ajoute un <div id="zone-message"> vide dans le <body> ;
    • dans exercice1.js, récupère cette zone avec document.getElementById() et remplis-la avec du contenu HTML via innerHTML.
  2. Ajoute un bouton dans le HTML, <button id="btn-hello">Dire bonjour</button>, puis dans le fichier JS, ajoute un écouteur click dessus pour afficher un message dans la console et dans la zone de message.
  3. Vérifie que ton fichier .js ne contient plus document.write() et que tout fonctionne en utilisant uniquement les manipulations de DOM.

Ces exercices te permettent de voir :

  • comment passer d’un script intégré à un script externe ;
  • pourquoi document.write() est à éviter dans le code moderne ;
  • comment modifier la page en JavaScript via le DOM.
